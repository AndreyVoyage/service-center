# ИТЕРАЦИЯ 2A: RBAC — Промежуточный Checkpoint

## ⚠️ ВАЖНО: Миграция существующего пользователя

На этом этапе `create: () => true` временно, чтобы создать admin пользователя.

## ✅ Verification 2A

### Шаг 1: Перезапустить `pnpm dev`

```bash
cd apps/cms
pkill -f "next dev" || true
pnpm dev
```

**Ожидаемый результат:** Сервер перезапущен без ошибок

### Шаг 2: Войти как dev@service.center (developer)

**Ожидаемый результат:**
- Видно меню Users
- Можно создать пользователя

### Шаг 3: Создать второго пользователя

1. Нажать "Create New" в коллекции Users
2. Заполнить:
   - Email: `admin@service.center`
   - Password: `password`
   - Role: **admin**
3. Сохранить

**Ожидаемый результат:**
- Пользователь admin@service.center создан
- В списке видны оба пользователя

## 📋 Чек-лист (Итерация 2A)

- [ ] Перезапущен `pnpm dev`
- [ ] Developer видит коллекцию Users
- [ ] Developer может создать пользователя
- [ ] Создан пользователь admin@service.center с ролью admin

## 🆘 Fallback Strategy

**Проблема:** Не видит коллекцию Users
**Решение:** Проверить `access: { admin: ... }` возвращает boolean
